# Integrations

Connect AAS to external tools and services.

## Claude Code

Enable file edits and code modifications via the Claude Code CLI.

**Install Claude:**
```bash
# Requires Claude Code CLI
npm install -g claude

# Or via Homebrew
brew install claude
```

**Connect:**
```bash
aas connect claude-code
aas config show | grep claude-code
```

**What it does:**
- Reads files and applies LLM-suggested edits
- Handles refactoring and code cleanup
- Integrates with your editor for code changes

**Example:**
1. Repository agent detects outdated code patterns
2. Claude Code analyzes the code
3. Agent applies fixes automatically
4. Verifies changes don't break tests

## OpenClaw

Delegate external tasks to OpenClaw (OpenAI, webhooks, custom services).

**Setup OpenClaw:**
```bash
# OpenClaw must be running separately
# See https://github.com/openai/openai-cookbook for setup

# Configure endpoint
aas config set llm.endpoint http://localhost:3001
```

**Connect:**
```bash
aas connect openclaw
```

**What it does:**
- Send notifications (Slack, email, webhooks)
- Call external APIs
- Run custom scripts
- Integrate with CI/CD systems

**Example:**
1. Health agent detects service down
2. Requests OpenClaw to send Slack alert
3. OpenClaw posts to #incidents channel
4. Oncall responds and takes ownership


## Manage Integrations

```bash
# Connect integration
aas connect claude-code

# Disconnect integration
aas disconnect openclaw

# View all connections
aas config show

# Test integration
aas test openclaw
```

## Troubleshooting

### Connection Failed

```bash
# Check if service is running
curl http://localhost:3001/health

# Check logs
aas logs follow | grep integration

# Validate configuration
aas validate-config
```

### Authentication Error

```bash
# Check credentials
aas config show

# Update API key
aas config set integrations.openclaw.api_key "new_key"

# Test connection
aas test openclaw
```

### Rate Limiting

If you hit rate limits:

```bash
# Increase detection interval
aas config set agents.repository.detection_interval 300s

# Reduce concurrent actions
aas config set execution.max_concurrent_actions 2

# Monitor usage
aas metrics
```

## Best Practices

1. **Start simple** — connect one integration at a time
2. **Test first** — use `aas test <integration>` before enabling
3. **Monitor logs** — watch for integration errors
4. **Set appropriate intervals** — avoid overwhelming external services
5. **Use API limits** — respect rate limits and quotas
