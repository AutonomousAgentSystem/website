# Changelog

All notable changes to AAS are documented here.

## [0.1.0] — 2026-06-21

### Added

- **Core Agent System** — Repository, Logs, Health, Metrics agents
- **Pattern Caching** — Learn solutions on first occurrence, reuse with 0 LLM calls on repeats
- **Recursive Self-Improvement (RSI)** — Dynamic confidence thresholds based on agent success rates
- **Multi-Provider LLM Routing** — Claude API, Hermes, Claude Code, OpenClaw with intelligent task routing
- **Staged Execution Pipeline** — Test → Validate → Execute → Verify with automatic rollback on failure
- **SQLite Memory Store** — Persistent learning database with pattern matching and prediction engine
- **Event-Driven Architecture** — Broadcast event bus for inter-agent communication
- **Web Dashboard** — Status monitoring, configuration UI, metrics visualization
- **Daemon Management** — Start/stop/restart background agents
- **Configuration** — Flexible JSON config with per-agent tuning
- **CLI Interface** — 25+ commands covering all operations
- **Docker Support** — Multi-stage Docker build with health checks
- **Kubernetes Helm Chart** — Production-ready deployment templates
- **Documentation** — Comprehensive guides, API reference, architecture docs

### Features

- ✅ Real execution (not simulation)
- ✅ Pattern caching (90% LLM savings at scale)
- ✅ Recursive self-improvement
- ✅ Multi-provider LLM support
- ✅ Rollback on failure
- ✅ Event bus for coordination
- ✅ SQLite persistence
- ✅ TUI dashboard
- ✅ Web API
- ✅ Systemd support
- ✅ Integration hooks (Claude Code, OpenClaw)

### Known Limitations

- Single-machine only (distributed coordination future work)
- Task and Trace agents are stubs (beta)
- Slack/email notifications coming soon
- Custom agent plugin system (future)
- No web UI authentication (local-only for now)

### Performance

- Cycle time (cache hit): ~500ms–2s
- Cycle time (cache miss): ~5–15s
- Memory footprint: ~50MB base + 1KB per pattern
- Cost model: 2 LLM calls per new issue type, 0 for repeats

## How to Update

### From Source

```bash
git pull origin main
cargo build --release
./target/release/aas --version
```

### Docker

```bash
docker pull aas:latest
docker run aas --version
```

### Kubernetes

```bash
helm repo update
helm upgrade aas aas/aas
```
